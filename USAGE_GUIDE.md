# SP-navigate 使用指南

## 📋 完整工作流程

本系统采用 **5 步工作流程**，确保地址准确性和规划方案的可执行性。

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 步骤 1   │ -> │ 步骤 2   │ -> │ 步骤 3   │ -> │ 步骤 4   │ -> │ 步骤 5   │
│ 准备数据 │    │ 地址验证 │    │ 上传修正 │    │ 生成规划 │    │ 查看结果 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

---

## 📥 步骤 1: 准备数据

### 方式 A: 下载标准模板（推荐）

1. 点击 **"📊 下载 Excel 模板"** 按钮
2. 打开下载的 `采样点标准模板.xlsx`
3. 按格式填写您的采样点数据

**模板格式说明：**

| 列名 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 序号 | 否 | 自动编号 | 1, 2, 3... |
| 地址 | **是** | 完整地址 | 长寿区凤城街道 XX 小区 |
| 小区名称 | 否 | 小区/单位名称 | XX 花园 |
| 备注 | 否 | 补充说明 | 近北门 |

**地址填写规范：**
- ✅ 正确：`长寿区凤城街道阳光小区`
- ✅ 正确：`长寿区菩提街道人民路 123 号`
- ❌ 错误：`阳光小区`（缺少区县）
- ❌ 错误：`重庆市长寿区...`（重复前缀）

### 方式 B: 直接上传已有数据

如果您已有格式正确的 Excel 文件，可直接上传。

**要求：**
- 必须包含 `地址` 列
- 地址格式：`区长治街道 + 详细地址`

---

## 🔍 步骤 2: 地址验证

### 自动验证

系统会自动调用 **高德地图 API** 验证每个地址：

1. **地理编码**：将地址转换为经纬度
2. **格式校验**：检查地址是否规范
3. **区域匹配**：确认地址在目标区域（长寿区）

### 验证结果

验证完成后，系统会显示：

| 状态 | 说明 | 处理方式 |
|------|------|----------|
| ✅ 匹配成功 | 地址有效，可正常规划 | 无需处理 |
| ❌ 匹配失败 | 地址无法识别 | 需要修正 |
| ⚠️ 区域不符 | 地址不在目标区域 | 需要确认 |

### 下载待修正地址表

如果有验证失败的地址：

1. 点击 **"📥 下载待修正地址表"**
2. 打开 Excel 文件，填写 `修正后地址` 列
3. 保存文件

**修正示例：**

| 原始地址 | 修正后地址 | 备注 |
|----------|------------|------|
| 阳光花园 | 长寿区凤城街道阳光花园 | 补充区县和街道 |
| 人民路 123 号 | 长寿区菩提街道人民路 123 号 | 补充区县和街道 |

---

## 📝 步骤 3: 上传修正后的地址表

### 上传修正文件

1. 点击 **"上传修正后的地址表"**
2. 选择已填写修正地址的 Excel 文件
3. 系统会自动合并修正数据

### 自动合并

系统会：
- 匹配 `原始地址` 列
- 使用 `修正后地址` 替换原地址
- 显示修正了多少个地址

### 重新验证（可选）

修正后可以选择重新验证，确保所有地址都有效。

---

## 🗺️ 步骤 4: 生成规划

### 配置参数

在侧边栏配置规划参数：

**策略选择：**
- 🏨 **隔夜住宿**：自动判断远近，远距离启用隔夜模式（推荐）
- 🚗 **单日往返**：全部当日返回
- 📍 **聚类分组**：按区域聚类

**约束条件：**
- 每日最大工时（小时）：推荐 8-10 小时
- 每日最大点数：推荐 3-8 个
- 每点停留时间（分钟）：推荐 15-30 分钟
- 隔夜距离阈值（公里）：推荐 60-100 公里

**起点配置：**
- 公司名称/起点：出发和返回地点
- 坐标（可选）：手动指定经纬度

### 开始规划

点击 **"▶️ 开始规划"**，系统会：

1. 计算所有点位之间的距离
2. 使用算法优化路线顺序
3. 按约束条件拆分为多天行程
4. 生成详细的规划方案

---

## 📊 步骤 5: 查看结果

### 规划概览

显示关键指标：
- 总点位数量
- 总天数
- 总行驶距离
- 总时间

### 每日行程详情

展开每天的行程，查看：

**基本信息：**
- 起点和终点

- 行程类型（单日往返/隔夜住宿）

**采样点列表：**
- 序号
- 名称
- 经纬度坐标

**住宿信息（隔夜行程）：**
- 住宿点位置
- 靠近的采样点

### 下载结果

**可下载的文件：**

| 文件 | 格式 | 用途 |
|------|------|------|
| 📊 Excel 结果 | .xlsx | 打印、分发 |
| 📄 JSON 结果 | .json | 程序处理 |

## 💡 常见问题

### Q1: 为什么有些地址验证失败？

**原因：**
- 地址不完整（缺少区县、街道）
- 地址名称错误或有错别字
- 地址太新，地图数据未更新

**解决方法：**
1. 下载待修正地址表
2. 补充完整地址信息
3. 上传修正表重新验证

### Q2: 如何选择合适的隔夜阈值？

**建议：**
- 80 公里：适合主城区及周边
- 60 公里：适合点位密集区域
- 100 公里：适合点位分散区域

**判断标准：**
- 单程驾驶时间 > 1.5 小时 → 建议隔夜
- 每日点位 > 5 个且距离远 → 建议隔夜

### Q3: 可以同时使用单日往返和隔夜住宿吗？

可以！选择 **"隔夜住宿"** 策略，系统会自动：
- 近距离点位 → 单日往返
- 远距离点位 → 隔夜住宿

### Q4: 如何修改已生成的规划？

1. 点击 **"🔄 重新规划"**
2. 调整参数（如每日点数、工时等）
3. 重新生成方案
