## 问题背景

### 应用场景

**重庆市长寿区水质采样路线规划**

### 地理特点

- 长寿区沿长江呈**带状分布**，东西长约 60km
- 长江阻隔，直线距离 1km 可能需要绕行 10km 过桥
- 点位分布分散，部分点位距离较远

### 当前问题

1. **质心法不适合带状分布**
   - 强行按圆形分组
   - 同一组的点位可能隔着长江

2. **异常点阈值过小**
   - 默认 5km
   - 导致大量点位被标记为"异常点"

3. **使用直线距离**
   - 未考虑长江阻隔
   - 未考虑绕行因素

## 方案 D: 改进的质心法（短期方案）

### 原理

在现有算法基础上调整参数和配置

### 改进点

1. **增加异常点阈值**: 5km → 20km
2. **使用驾驶距离**: 启用高德 API 路径规划
3. **改用链式法**: 更适合带状分布

### 配置

```yaml
strategy: cluster
cluster_method: chain  # 链式法
outlier_threshold_km: 20.0  # 异常点阈值 20km
use_amap_route: true  # 启用高德路径规划
```

### 优点

- ✅ 改动最小
- ✅ 快速见效
- ✅ 无需开发

### 缺点

- ❌ 效果提升有限
- ❌ 仍不适合复杂分布

### 实施状态

✅ **已完成**

## 方案 A: DBSCAN 密度聚类（中期方案）

### 原理

基于密度的聚类，自动发现任意形状的簇

### 算法

```python
from sklearn.cluster import DBSCAN
import numpy as np

# 准备坐标数据（弧度制）
coords = np.radians([[p.lat, p.lng] for p in points])

# DBSCAN 聚类
db = DBSCAN(eps=0.05, min_samples=3, metric='haversine')
labels = db.fit_predict(coords)

# labels 就是每个点位的分组编号
# -1 表示异常点
```

### 参数

| 参数 | 说明 | 建议值 |
|------|------|--------|
| `eps` | 邻域半径（弧度） | 0.03-0.05（约 3-5km） |
| `min_samples` | 最小点数 | 3-5 |

### 优点

- ✅ 适合带状、沿江分布
- ✅ 自动确定分组数量
- ✅ 自动识别异常点
- ✅ 不需要预设分组数

### 缺点

- ❌ 需要调参
- ❌ 需要安装 sklearn

### 实施状态

🔄 **开发中**

## 方案 C: 道路网络聚类（长期方案）

### 原理

使用高德 API 获取实际驾驶距离，基于路网距离聚类

### 步骤

1. **调用高德 API** 获取所有点位间的驾驶距离矩阵 (n² 次调用)
2. **使用驾驶距离** 而非直线距离进行聚类
3. **考虑过桥、绕行** 等因素

### 优点

- ✅ 最准确
- ✅ 反映真实行驶情况
- ✅ 考虑长江阻隔

### 缺点

- ❌ API 调用次数多 (n² 次)
- ❌ 规划时间长
- ❌ 成本较高

### 实施状态

⏳ **待实施**

## 方案 B: 地理围栏 + 聚类（备选方案）

### 原理

先按地理区域粗分，再在区域内聚类

### 步骤

1. **按街道/镇划分**（长寿区有 7 个街道、5 个镇）
2. **每个街道内使用聚类**
3. **相邻街道可以合并**

### 优点

- ✅ 符合行政区划
- ✅ 便于管理
- ✅ 减少跨区行驶

### 缺点

- ❌ 需要行政区划数据
- ❌ 边界点位处理复杂

### 实施状态

⏳ **待实施**

### DBSCAN 参数换算

```

eps = 5 / 6371 ≈ 0.000785 弧度
```
