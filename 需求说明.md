# 多点位路线规划需求说明

## 基本信息
- **出行方式**: 驾车
- **点位格式**: 地址名称
- **地图 API**: 高德地图 Web服务
- **API Key**: b6410cb1a118bad10e6d1161d6e896f7

## 约束条件
- 每天导航总时间不超过 **6小时**
- 超出6小时的路线自动拆分为多天

## 路线规则
- 有**固定起点**（每天从同一地点出发）
- 有**固定返回点**（每天返回同一地点）
- 每天路线为：起点 → 若干途经点 → 返回点

## 处理逻辑
1. 对所有点位进行地理编码（地址 → 坐标）
2. 调用高德驾车路径规划 API 计算所有点之间的距离/时间矩阵
3. 用 TSP 算法求解最优访问顺序
4. 按6小时限制拆分为多天路线
5. 每天路线包含往返时间（起点出发 + 返回终点）
6. 生成每天的可视化地图和汇总报告
